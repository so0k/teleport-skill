Reusable lesson components

These are the shared building blocks for every lesson. Styles live in course.css. Copy the HTML snippet as-is so all lessons stay consistent. This page is for authoring, not for learners.

1. Beat markers (Why → What → How)

Every lesson opens its three beats with these.

Why · the problem What · the solution approach How · the mechanics

2. 🔴 → 🟢 Regression locked in

Ends every lesson from Lesson 4 onward. States the failure this lesson found (red) and the eval that now freezes it (green). This is how the course models test-first discipline.

🔴 → 🟢   The regression we locked in

Red: what failed, and how it was measured.

Green: the eval that now asserts the fix — and stays in the suite forever.

3. ⏱️ Worth it? (ROI caveat)

Use wherever a lesson asks the learner to spend real effort. Keeps the honest question alive: does this ancillary work pay for itself, × how many agents reuse the skill?

⏱️ Worth it? The cost of this step, weighed against just doing the work — and who keeps benefiting.

4. Tension line, callouts, diagram, quiz

Already in use; see course.css classes: .tension, .note / .warn / .key, .diagram (with .hi / .hi2 spans), and .quiz with <details> for retrieval practice.

Two recurring threads run on top of these: progressive discovery (each lesson sources more accurate data than the last) and the two boxes above (red→green + worth-it).
